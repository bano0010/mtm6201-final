## Process

I started this project by using the mockups, wireframes, and design elements I created in my User Experience Design course. These designs helped me plan the layout, colors, spacing, and responsive structure of the website before building it in code.

I then created a three-page website using Bootstrap as the main framework. I customized the design by using CSS variables for colors and added my own styling in the main stylesheet. I worked on making the layout responsive for desktop, tablet, and mobile screen sizes.

I built the Home page to introduce the website and show featured products, the Shop page to display product listings and category filters, and the About page to explain the story, mission, vision, and values of the brand.

## Challenges and Solutions

One challenge I faced was making the website match my Figma layouts across different screen sizes. Sometimes spacing, image sizes, and text sizes looked correct on desktop but not on tablet or mobile. I fixed this by testing each page carefully in responsive view and adjusting layout, image ratios, spacing, and font sizes in my CSS media queries.

Another challenge was keeping the navigation and footer consistent across all pages. I solved this by reusing the same structure and styles so the website would feel unified.

I also had some issues with the mobile navigation menu and responsive product spacing. I resolved these by checking my Bootstrap setup, making sure the proper JavaScript file was linked, and refining the mobile CSS.

## What I Learned

By creating this project, I learned how to turn a Figma design into a working responsive website using Bootstrap and custom CSS. I improved my understanding of responsive design, layout structure, spacing, navigation, accessibility basics, and testing across multiple screen sizes.

I also learned how important it is to organize code clearly, keep styles consistent, and solve design problems step by step during development.
